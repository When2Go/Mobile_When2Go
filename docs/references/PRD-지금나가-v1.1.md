# PRD: "지금 나가?" v1 (MVP)

> 실시간 출발 타이밍 판단 모바일 앱
> 작성일: 2026-04-03 | 최종 수정: 2026-04-08 | 팀: 지금이니 | MVP 목표: 2026년 6월 말

---

## 1. 프로젝트 개요

### 1.1 문제 정의

사람들은 길을 몰라서 늦는 것이 아니라, **언제 출발해야 하는지를 정확히 판단하지 못해** 지각하거나 불필요하게 일찍 도착하는 문제를 겪는다.

- 대중교통 이용자는 환승, 배차 간격, 막차 시간 등 여러 변수 속에서 "지금 출발해도 되는지"를 스스로 판단해야 한다.
- 기존 지도 앱은 경로와 소요시간만 제공할 뿐, **출발 시점 판단은 여전히 사용자에게 맡겨져 있다.**
- '앗차!' 서비스가 유사한 아이디어로 출발 타이밍을 계산하고 있으나, **막차 관련 기능만 제공**하는 한계가 있다.

**사용자 조사 결과** (37명 대상 설문):
- 응답자의 **94.2%** (35/37명)가 "언제 출발해야 할지" 때문에 스트레스를 받은 경험이 있다고 응답
- 응답자의 **88.6%** (33/37명)가 도착 시간과 실시간 교통 상황 기반 출발 안내 서비스 사용 의향이 있다고 응답

### 1.2 솔루션

"지금 나가?"는 사용자의 도착 목표 시간, 현재 위치, 실시간 교통 상황을 바탕으로 **언제 출발해야 하는지를 계산하고, 단계별 알림으로 출발을 안내하는 모바일 앱**이다.

- 경로 제공이 아닌 **출발 판단**에 집중
- MVP는 **규칙 기반 로직 + 실시간 교통 API** 중심으로 빠르게 구현
- 이후 AI를 보조적으로 활용하여 자연어 일정 입력, 개인 패턴 보정 등 고도화

### 1.3 타겟 사용자

전국 대중교통 이용자. 특히 약속, 통학, 출근 등 **정해진 시간에 맞춰 이동해야 하는 사람.**

### 1.4 핵심 차별점

| 항목 | 카카오맵/네이버 | 앗차! | "지금 나가?" |
|------|--------------|-------|------------|
| 핵심 | 경로 안내 | 막차 타이밍 | **출발 타이밍 판단** |
| 실시간 반영 | 경로 소요시간 | 막차 시간 | 교통 변화 시 재계산 |
| 개인화 | 즐겨찾기 | 없음 | O (패턴 학습, 고도화) |
| 잠금화면 | X | X | O (Live Activity) |
| 알림 | X | 막차 알림 | 단계별 출발 알림 |

기존 앱은 모든 사용자에게 동일한 정적 정보를 제공하지만, "지금 나가?"는 실시간 교통 상황을 반영한 출발 시점 판단을 제공하며, 이후 개인 이동 패턴 학습을 통해 **사용할수록 정확해지는 개인화된 출발 타이밍**으로 고도화한다.

## 4. 사용자 플로우

### 4.1 MVP 기본 플로우 (일정 등록 → 출발 → 도착)

```
앱 실행 (메인 화면: 네이버맵 + 현재 위치)
  ↓
목적지 입력 (검색)
  ↓
도착 시간 설정
  ↓
경로 옵션 선택 (최적경로 / 최소환승 / 지하철 우선 / 버스만)
  ↓
출발 시간 계산 결과 화면
(지도: 현재 위치 + 첫 탑승 위치 마커 / 텍스트 경로 안내 / 대중교통 도착 예정 시간)
  ↓
예약 리스트에 등록
  ↓
[다단계 재계산: 1시간 전 / 30분 전 / 20분 전 ODsay 재호출 → 출발 시점 보정]
  ↓
[푸시] 출발 10분 전 알림
  ↓
[푸시] 출발 시점 알림
  ※ 등록 시점에 이미 지각이면 위 2건 대신 즉시 "지금 바로 출발하세요" 알림 1건 발송
  ↓
사용자 [출발했어요] 버튼 탭
  ↓
사용자 [도착했어요] 버튼 탭
  ↓
Trip 완료
```

### 4.2 고도화 플로우 (5월~)

```
[기본 플로우에 추가]
  ↓
사용자 [출발했어요] 버튼 탭
  ↓
잠금화면 위젯 활성화
(캐릭터 프로그레스 바 + 실시간 위치/경로 + 대중교통 도착 예정)
GPS 실시간 트래킹 시작
  ↓
사용자 [도착했어요] 버튼 탭
  ↓
Trip 완료 → 예측 오차 기록 → 전면 광고 1회
```

### 4.3 즐겨찾기 플로우 (고도화)

```
사용자 페이지 → 자주 가는 곳
  ↓
목적지 저장 (목적지(필수), 시간(옵션), 별명(옵션))
  ↓
메인 화면에서 즐겨찾기 선택 → 바로 시간 설정으로 이동
```

### 4.4 자연어 입력 플로우 (완성 단계)

```
음성/텍스트로 자연어 입력 (ex. "내일 2시 강남역 도착")
  ↓
LLM이 목적지/시간 자동 파싱
  ↓
결과 확인 → 출발 시간 계산 플로우로 이동
```

## 5. 시스템 아키텍처

### 5.1 기술 스택

| 레이어 | 기술 | 단계 | 비고 |
|--------|------|------|------|
| Mobile | React Native + Expo (TypeScript) | MVP | Expo Router |
| iOS 위젯 | Swift / SwiftUI | 고도화 | Live Activity, Dynamic Island |
| Android 위젯 | Foreground Service | 고도화 | Ongoing Notification |
| Map | 네이버맵 Native SDK | MVP | @mj-studio/react-native-naver-map |

## 6. 수익 모델

### 6.1 v1: 전체 무료 + Google AdMob 광고

초기에는 무료 기반으로 사용자 유입과 서비스 검증을 우선한다. 핵심 흐름을 방해하지 않는 선에서 가벼운 광고를 배치한다.

| 광고 위치 | 형식 | 시점 | 단계 |
|----------|------|------|------|
| 홈 화면/결과 화면 | 배너 (소규모) | 앱 사용 중 상시 | MVP |
| Trip 완료 | 전면(Interstitial) 팝업 | Trip 도착 완료 시 1회 | 고도화 |
| 잠금화면 위젯 하단 | 배너 (소규모) | 위젯 활성 중 상시 | 고도화 |

### 6.2 v2: 프리미엄 구독

일정 연동 및 반복 패턴 학습과 같은 핵심 차별화 기능을 프리미엄 서비스로 전환. 초기에는 일정 기간 무료 제공 후 유료 전환을 유도한다.

| 프리미엄 기능 | 설명 |
|-------------|------|
| 캘린더 자동 연동 | 음성 입력과 연계하여 캘린더 일정 자동 등록 |
| 개인화 반복 패턴 학습 | 개인별 걷기 속도, 준비 시간, 이동 성향 학습 |

### 6.3 v2: 맥락 기반 푸시 알림 광고

사용자의 목적지 및 이동 맥락을 반영한 맥락 기반 푸시 알림 광고. 특정 목적지 설정 시 인근 음식점, 카페 등을 추천 형태의 알림으로 자연스럽게 전달한다.

### 6.4 초기 자금 확보

대규모 투자유치보다 **교내 창업 지원 프로그램, 공모전, 정부 지원사업** 등을 활용해 MVP 개발과 테스트에 필요한 자금을 확보한다. 실제 사용자 반응과 서비스 활용 데이터를 확보한 뒤 다음 단계 자금조달을 검토한다.
