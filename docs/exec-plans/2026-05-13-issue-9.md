## 목표

`app/schedule.tsx`를 빈 placeholder에서 실제 일정 화면으로 채운다. 시안(`../design/src/app/pages/Schedule.tsx`) 구조를 RN/NativeWind로 옮기되, 데이터·시트 상세는 mock 수준에서 끝낸다. 핵심은 **상단 미니 캘린더 + 선택 날짜의 예약 리스트 + 카드에 "xx:xx에 갱신됨" 노출**이다.

## 범위

- ✅ 상단 미니 캘린더 (월간, 오늘 강조 + 선택 날짜 강조)
- ✅ 선택 날짜의 예약 리스트 (mock 3건)
- ✅ 각 카드: 출발·도착 시간, 목적지, 경로, 상태 뱃지, 갱신 시각(F-M06)
- ✅ 빈 상태 (선택한 날짜에 예약이 없을 때 안내 + 새 일정 추가 버튼)
- ✅ 카드 좌측 스와이프 → 삭제 (mock, 로컬 state)
- ✅ 카드 탭 → bottom sheet 상세 (시안 수준의 정보만)
- ✅ Light / Dark 모드
- ✅ MobileLayout 헤더의 "일정" 타이틀 유지 (이미 매핑됨), 우측 "새 일정" 액션은 페이지 내부 별도 헤더 또는 리스트 위 영역으로

## 범위 밖

- Trip API · 실 데이터 동기화 (별도 이슈)
- 외부 캘린더 라이브러리 도입 — 시안 그대로 grid-7 수제 캘린더로
- 새 일정 추가 화면 라우팅 실제 동작 (`/search` 푸시까지만)
- 광고 슬롯 (Schedule 화면은 슬롯 매핑 없음)
