# /quality — 품질 등급 재평가 워크플로우

현재 프로젝트 코드를 전체 분석해서 `docs/QUALITY_SCORE.md`를 업데이트.
각 영역별로 근거와 함께 등급(A~F)을 매기고, 이전 등급과 비교해서 변화(↑↓→) 표시.

---

## 평가 영역

| 영역 | 평가 기준 |
|------|---------|
| 컴포넌트 재사용성 | 공용 컴포넌트 활용 비율, 중복 구현 |
| 디자인 시스템 일관성 | 하드코딩 색상 수, 토큰 사용률, 다크모드 누락 |
| 상태 관리 | 상태 분류 준수, 과도한 전역 상태 |
| API 패턴 | 타입 통일, 에러 처리 일관성 |
| 타입 안전성 | any 사용 수, 타입 누락 |
| 코드 품질 | 매직 넘버, 중첩 삼항, Props Drilling |
| 테스트 커버리지 | 테스트 파일 존재 여부 |
| 접근성 | accessibilityLabel 등 |
